开始
git config --global user.name “panjia”； 设置你的名字 （必须）
git config --global user.email "panjia_vip@163.com"; 设置你的邮箱 （必须）
git init 初始化，创建一个版本库

查看状态
git status 查看修改状态
git diff 查看修改详情
git log 查看历史记录 --pretty=oneline（简化输出）
git reflog 查看退回版本历史操作记录
cat 文件名 查看文件内容
git config --list 查看config信息

git config -e    编辑core origin
vim ~/.gitconfig 编辑config

操作阶段
git add index.php 添加一个或多个待提交文件用空格隔开
git commit -m “测试” 提交文件到版本库

退回历史
说明：一个 head^ 代表退回一个版本，退回多个用^^ 或 head~100
git reset --hard head^ 退回上个版本
git reset --hard <commit ID> 根据commit ID 退回


退回操作
未添加到暂存区前：
git checkout -- <file_name> 丢弃工作区的操作
说明：丢弃工作区的操作会退回到暂存区的走回一次 或 版本库的最新版本 特别注意--后面必须跟空格

已添加到暂存区后：
git reset head <file_name> 撤销刚刚添加的文件

删除操作
git rm <file_name> rm 后面跟上文件名，后面直接跟commit

创建SSH key
说明：首先用户目录看一下有没有.ssh文件夹，进入里面看看有没有id_rsa 和 id_rsa.pub 两个文件，如果没有就创建一个
命令：ssh-keygen -t rsa -C "自己的邮箱" 生成id_rsa 和 id_rsa.pub

远程仓库
连接远程
git remote add origin 连接到仓库地址url

推送远程
git push -u origin master 推送到远程仓库master分支 注意： 第一次推需要github账号密码
git push origin master 推送最新的修改到master分支 注意：如果出现邮箱限制问题，请在github上设置email限制

克隆远程
git clone <gitorigin_url> 返回最新的github代码
git clone -b dev <gitorigin_url> 返回对应分支

抓取合并
git pull origin <远程name:本地name>

查看远程
git remote 查看远程分支 加上-v 查看详细信息 注意：fetch为可抓取分支，push为可推送分支

分支管理
创建分支
git branch <name> 创建分支
git checkout -b dev 创建并切入dev分支 -b参数创建并切入

查看所有分支
git branch 查看所有分支，带星号的为当前分支
git branch -av 查看全部分支，包括远程分支

切换分支
git checkout <name> 切换到对应分支

合并分支
git merge <name> 合并对应分支
git merge --no-ff -m '说明' branch_name 合并分支并说明

删除分支
git branch -d <name> 删除分支 用合并分支后 -d参数 删除
git branch -D <name> 强制删除分支，用于合并前 -D参数 删除

查看分支线
git log --graph --pretty=oneline --abbrev-commit

隐藏工作内容
git stash 会隐藏工作区你所修改的操作
git stash pop 恢复隐藏工作去内容，删除stash里的备份
git stash apply 恢复隐藏工作区内容，不删除 stash里的备份
git stash drop 删除隐藏工作区内容

创建远程分支
首先：git branch <name> 在本地创建一个分支
然后：git push origin <name> 推送到远程origin 这样就远程仓库就创建成功了

删除远程分支
git push origin ：<name> 空的分支，分支名字前加：
git push origin --delete <name> 可删掉远程分支

建立分支关联
git branch --set-upstream dev origin/dev

标签管理
创建标签
git tag <tagname> 创建一个标签
git tag <tagname> commit ID 历史记录打标签

查看标签
git tag 查看所有标签
git show <tagname> 查看标签内容

删除标签
git tag -d <tagname> 删除对应标签

推送标签
git push origin <tagname> 推送对应标签到远程
git push origin --tags 推送全部标签到远程

删除远程标签
首先：git tag -d <tagname> 删除本地标签
然后：git push origin :refs/tags/<tagname> 推送删除远程标签

添加忽略文件
.gitignore 注意此文件，忽略文件，创建时可用命令来创建
注意：如果想添加的文件正好在忽略名单中就用git add -f <name> 强制添加
git check-igonre -v app.class 查看忽略文件错误信息
